# infotecs_intern
Клиент серверное приложение для передачи файлов по сети

## Требования для сборки из исходного кода:

* ОС Linux;
* Компилятор GCC;
* Система сборки CMake;
* Инструмент сборки Make.


> **Для установки данных пакетов используйте следующие команды:**
>
> ```bash
> sudo apt update
> sudo apt install build-essential cmake g++
> ```

## Сборка проекта

```bash
mkdir build && cd build
cmake ..
cmake --build .
```
После сборки проекта в папке build будут созданы:
* Исполняемые файлы server_app, client_app;
* Динамические библиотеки libfileop.so, libprogress.so.

## Запуск
### Запуск сервера
Опциональные флаги:
* -p <номер порта> - указывает порт на котором сервер будет прослушивать подключения(по умолчанию порт = 1313).
> Пример: ```./server_app```


### Запуск клиента
Обязательные флаги:
* -src <источник> - указывает откуда брать файл
* -dst <получатель> - указывает куда будет загружен файл

Опциональные флаги:
* -p <номер порта> - указывает порт сервера к которому нужно подключиться(по умолчанию порт = 1313).
* -chunk <размер порции> - задает размер порции для передачи данных.

Формат команды запуска приложения клиента:
* для загрузки файла на сервер(upload):  
**```./client_app -src <путь к файлу> -dst <IP-адрес>:<путь к файлу>```**
> Пример(upload): **```./client_app –src ./file1 -dst 127.0.0.1:/tmp/file1 ```**
* для загрузки файла с сервера(download):  
**```./client_app -src <IP-адрес>:<путь к файлу> -dst <путь к файлу>```**
> Пример(download): **```./client_app -src 127.0.0.1:/tmp/file2 –dst ./file2 ```**

## Выполненные требования

### Обязательные требования

* ОС Linux;
* Язык программирования С++ (Стандарт С++17);
* Взаимодействие осуществляется по TCP/IP (IPv4);
* Серверная часть поддерживает возможность обслуживания нескольких соединений одновременно;
* Соединения обслуживаются с помощью потоков;
* Есть возможность получения файла от сервера клиентом;
* Настройка передачи файла производится с помощью аргументов "-src (удалённый источник файла)" и "-dst (локальный получатель файла)";
* Передача файла осуществляется фиксированными порциями определённого размера, размер порции определяется параметром запуска приложения клиента "-chunk (размер порции)" или значением по умолчанию;
* Сетевой порт задаётся как параметр запуска у сервера и клиента аргументом "-p (порт)" или значением по умолчанию;
* При передаче или приеме файла на клиентской стороне показывается прогресс передачи в процентах и с указанием количества переданных байт в формате <передано xx байт>/<общий размер файла yy байт>;
* На диск сохраняется только полностью переданный файл.

### Необязательные требования

* Есть возможность передачи файла от клиента серверу;
* Настройка передачи файла производится с помощью аргументов "-src (локальный источник файла)" и "-dst (удалённый получатель файла)"
* Функция печати прогресса реализована в виде динамической библиотеки, использовано позднее связывание;
* Работа с файловой системой реализована в виде динамической библиотеке, использовано ранее связывание;
* На клиентской стороне предусмотрена возможность прерывания операции передачи/приёма файлов пользователем(Ctrl + C) с коректным закрытием TCP-соединения;
* На серверной стороне предусмотрена возможность прерывания операции передачи/приема файлов сервером(Ctrl + C) с корректным закрытием TCP-соединения.
